## 背景

公司内部目前使用的IP转地区码的服务是基于[ipip.net](https://www.ipip.net/)的，之前都是在PHP里使用，可以直接读取物理机上的IP库文件，所有机器上的IP库文件会由另外一个脚本机器定时统一更新。

但最近我们的技术栈增加了 Golang，但我们的服务都是在容器里运行，不能直接读取物理机上的IP库文件，所以在寻找一个更好的解决方案。

## 前期尝试

[ipip.net](https://www.ipip.net/) 官方提供了一个 Golang 包
https://github.com/ipipdotnet/ipdb-go

使用后发现，输入的结果并不是我们所期望的 IP地址码。

## API服务

IP转地区码

```bash
curl -X GET "https://host.com/get?ip=211.107.26.1"

{
    "code": 200,
    "message": "ok",
    "data": "1_410_000000",
    "extra": {}
}
```

IP转详细信息

```bash
curl -X GET "https://host.com/get/detail?ip=114.10.134.249"

{
    "code": 200,
    "message": "ok",
    "data": {
        "ip_code": "1_360_000000",
        "country_name": "印度尼西亚",
        "region_name": "印度尼西亚",
        "city_name": "",
        "location_desc": "印度尼西亚",
        "china_admin_code": "000000",
        "continent_code": "AP",
        "country_code": "ID",
        "europe_union": "",
        "idd_code": "62",
        "isp_domain": "",
        "latitude": "0.10974",
        "longitude": "113.917397",
        "owner_domain": "",
        "timezone": "Asia/Jakarta",
        "utc_offset": "UTC+7"
    },

}
```
